## Overview

This document outlines the technical implementation phases for the BalanceX MVP enhancement. It translates the product requirements from `MVP_SPECIFICATION.md` into actionable development tasks.

### Current State
- Next.js 16.1.4 + React 19 + TypeScript
- UI: shadcn/ui + Tailwind CSS 4
- State: Zustand with localStorage persistence
- ~80% of UI screens exist
- No authentication
- No backend database

### Target State
- Authentication with role-based permissions (Owner, Accountant, Sales)
- PostgreSQL database with Prisma ORM
- Refactored daily entry with credit customer linking
- Screenshot upload and verification workflow
- Variance validation (warning/block logic)
- Complete calculation engine for all business logic

```typescript
// /src/lib/permissions.ts
export type Role = 'owner' | 'accountant' | 'sales'

export const PERMISSIONS = {
  // Daily Entry
  DAILY_ENTRY_CREATE: ['owner', 'accountant', 'sales'],
  DAILY_ENTRY_EDIT_OWN: ['owner', 'accountant', 'sales'],
  DAILY_ENTRY_EDIT_SUBMITTED: ['owner', 'accountant'], // + time check for accountant
  DAILY_ENTRY_DELETE: ['owner'],

  // Credit
  CREDIT_CUSTOMER_VIEW: ['owner', 'accountant', 'sales'],
  CREDIT_CUSTOMER_MANAGE: ['owner', 'accountant'],
  CREDIT_LIMIT_OVERRIDE: ['owner'],

  // Screenshot
  SCREENSHOT_UPLOAD: ['owner', 'accountant'],
  SCREENSHOT_DELETE: ['owner'],

  // Bank
  BANK_VIEW: ['owner', 'accountant'],
  BANK_MANAGE: ['owner', 'accountant'],

  // Wallet
  WALLET_VIEW: ['owner', 'accountant', 'sales'],
  WALLET_TOPUP: ['owner', 'accountant'],

  // Reports
  MONTHLY_REPORT: ['owner', 'accountant'],

  // Settings
  USER_MANAGEMENT: ['owner'],
  DATA_EXPORT: ['owner', 'accountant'],
  DATA_CLEAR: ['owner'],
}

// Check basic permission
export function hasPermission(role: Role, permission: keyof typeof PERMISSIONS): boolean

// Check if user can edit specific entry
export function canEditEntry(
  role: Role,
  entryUserId: string,
  currentUserId: string,
  entryDate: Date,
  isSubmitted: boolean
): boolean {
  if (role === 'owner') return true
  if (role === 'accountant') {
    if (!isSubmitted) return true
    // Can edit submitted entries from last 7 days
    const daysDiff = differenceInDays(new Date(), entryDate)
    return daysDiff <= 7
  }
  if (role === 'sales') {
    // Can only edit own unsubmitted entries
    return entryUserId === currentUserId && !isSubmitted
  }
  return false
}
```

#### Credit Limit Check Logic

```typescript
async function checkCreditLimit(customerId: string, newAmount: number): Promise<{
  allowed: boolean
  requiresOverride: boolean
  currentOutstanding: number
  creditLimit: number | null
  newTotal: number
  exceedsBy: number
}> {
  const customer = await getCustomer(customerId)
  const currentOutstanding = await getOutstandingBalance(customerId)
  const newTotal = currentOutstanding + newAmount

  if (!customer.creditLimit) {
    return { allowed: true, requiresOverride: false, ... }
  }

  if (newTotal > customer.creditLimit) {
    return {
      allowed: false,
      requiresOverride: true, // Owner can override
      currentOutstanding,
      creditLimit: customer.creditLimit,
      newTotal,
      exceedsBy: newTotal - customer.creditLimit
    }
  }

  return { allowed: true, requiresOverride: false, ... }
}
```

### 2.3 Cash Drawer Calculations

**Status:** Not Started

#### Formula (from MVP Spec)

```
Expected Closing = Opening Balance
                 + Total Cash Sales (all categories)
                 + Cash Settlements Received (today)
                 - Bank Deposits (today)
                 - Wallet Top-ups from Cash (today)

Variance = Actual Closing - Expected Closing
```

#### Detailed Breakdown

```typescript
interface CashDrawerCalculation {
  opening: number                    // Manual input

  // Additions
  cashSales: {
    dhiraaguBills: { consumer: number, corporate: number }
    retailReload: { consumer: number, corporate: number }
    wholesaleReload: { consumer: number, corporate: number }
    sim: { consumer: number, corporate: number }
    usim: { consumer: number, corporate: number }
    total: number
  }
  cashSettlements: number            // Credit customer payments in cash (today)

  // Deductions
  bankDeposits: number               // Manual input
  walletTopupsFromCash: number       // Sum of topups where source = 'cash'

  // Results
  closingExpected: number            // Calculated
  closingActual: number              // Manual input
  variance: number                   // Calculated
}

function calculateCashDrawerExpected(data: CashDrawerCalculation): number {
  return data.opening
       + data.cashSales.total
       + data.cashSettlements
       - data.bankDeposits
       - data.walletTopupsFromCash
}
```

### 2.4 Wallet Calculations

**Status:** Not Started

#### Formula (from MVP Spec)

```
Expected Closing = Opening Balance
                 + Top-ups (today)
                 - Reload Sales Used (Retail + Wholesale)

Opening Balance = Previous day's Actual Closing
                  OR Initial setup value (first time)
                  OR Manual entry (if previous day missing)

Reload Sales Used = Retail Reload (Consumer Cash + Consumer Transfer + Corporate Cash + Corporate Transfer)
                  + Wholesale Reload (Consumer Cash + Consumer Transfer + Corporate Cash + Corporate Transfer)

Variance = Actual Closing - Expected Closing
```

#### Detailed Breakdown

```typescript
interface WalletCalculation {
  opening: {
    balance: number
    source: 'previous_day' | 'initial_setup' | 'manual'
    previousDate?: Date
  }

  topups: {
    fromCash: number
    fromBank: number
    total: number
  }

  reloadSalesUsed: {
    retailReload: { consumer: number, corporate: number, total: number }
    wholesaleReload: { consumer: number, corporate: number, total: number }
    total: number
  }

  closingExpected: number
  closingActual: number
  variance: number
}

function calculateWalletExpected(data: WalletCalculation): number {
  return data.opening.balance
       + data.topups.total
       - data.reloadSalesUsed.total
}
```

| Wallet variance > MVR 500 | **HARD BLOCK** | Red |

#### Technical Approach

```typescript
interface ValidationResult {
  canSubmit: boolean
  hasWarnings: boolean
  hasBlocks: boolean
  creditBalanced: boolean
  cashVariance: { value: number, status: 'ok' | 'warning' | 'block' }
  walletVariance: { value: number, status: 'ok' | 'warning' | 'block' }
  messages: { type: 'warning' | 'block', message: string }[]
}

function validateDailyEntry(entry: DailyEntry): ValidationResult {
  const result: ValidationResult = {
    canSubmit: true,
    hasWarnings: false,
    hasBlocks: false,
    creditBalanced: true,
    cashVariance: { value: 0, status: 'ok' },
    walletVariance: { value: 0, status: 'ok' },
    messages: []
  }

  // Credit balance check
  const gridConsumerCredit = entry.categories.dhiraaguBills.consumerCredit
  const gridCorporateCredit = entry.categories.dhiraaguBills.corporateCredit
  const sectionConsumerCredit = sumCreditSales(entry.creditSales, 'consumer')
  const sectionCorporateCredit = sumCreditSales(entry.creditSales, 'corporate')

  if (gridConsumerCredit !== sectionConsumerCredit ||
      gridCorporateCredit !== sectionCorporateCredit) {
    result.creditBalanced = false
    result.canSubmit = false
    result.hasBlocks = true
    result.messages.push({
      type: 'block',
      message: 'Credit sales do not match grid totals'
    })
  }

  // Cash variance check
  const cashVar = Math.abs(entry.cashDrawer.actual - entry.cashDrawer.expected)
  result.cashVariance.value = cashVar
  if (cashVar > 500) {
    result.cashVariance.status = 'block'
    result.canSubmit = false
    result.hasBlocks = true
    result.messages.push({
      type: 'block',
      message: `Cash variance MVR ${cashVar.toFixed(2)} exceeds MVR 500 limit`
    })
  } else if (cashVar > 0) {
    result.cashVariance.status = 'warning'
    result.hasWarnings = true
    result.messages.push({
      type: 'warning',
      message: `Cash variance: MVR ${cashVar.toFixed(2)}`
    })
  }

  // Wallet variance check (same logic)
  // ...

  return result
}
```

## Appendix A: All Calculation Formulas

### Daily Entry Calculations

```
Cash Drawer Expected = Opening
                     + Cash Sales (all categories)
                     + Cash Settlements (today)
                     - Bank Deposits
                     - Wallet Top-ups from Cash

Cash Variance = Actual - Expected

Wallet Expected = Opening
                + Top-ups (today)
                - Reload Sales (Retail + Wholesale)

Wallet Variance = Actual - Expected

Daily Revenue = Dhiraagu + Retail + Wholesale + SIM + USIM

Consumer Total = Sum of Consumer (Cash + Transfer + Credit) across all categories
Corporate Total = Sum of Corporate (Cash + Transfer + Credit) across all categories

Cash Total = Sum of Cash across all categories
Transfer Total = Sum of Transfer across all categories
Credit Total = Dhiraagu Bills Credit (Consumer + Corporate)
```

### Credit Calculations

```
Customer Outstanding = Sum(Credit Sales) - Sum(Settlements)

Consumer Outstanding = Sum of Consumer customers' outstanding
Corporate Outstanding = Sum of Corporate customers' outstanding
Total Outstanding = Consumer + Corporate
```

### Bank Calculations

```
Running Balance = Previous Balance + Deposit - Withdrawal
Current Balance = Latest Running Balance OR Opening (if no transactions)
Monthly Deposits = Sum of Deposits in month
Monthly Withdrawals = Sum of Withdrawals in month
```

### Monthly Calculations

```
Monthly Revenue = Sum of Daily Revenue
Days Complete = Count where (Submitted AND Screenshot AND Verified)
Average Daily = Monthly Revenue / Days with entries
Total SIM = Sum of daily SIM quantities
Total USIM = Sum of daily USIM quantities
```
